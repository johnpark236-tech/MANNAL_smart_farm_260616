만날 농사 앱 인수인계 개요

스마트팜 작업자를 위한 Android APK 프로젝트의 현재 구조, 기능, 빌드 위치, 협업 포인트를 정리한 문서입니다.

작성 기준: 2026-06-17
현재 앱 버전: 1.4.0 / versionCode 14
앱 표시명: 만날 농사
패키지명: com.mannal.agri

1. 한 줄 요약

만날 농사는 농사꾼이 스마트폰에서 시세, 농사 날씨, 영농일지를 빠르게 확인하고 음성으로 날씨를 듣거나 영농일지를 기록할 수 있게 만든 네이티브 Android 앱입니다.

2. 저장 위치

항목위치설명
프로젝트 루트 C:\Users\Admin\OneDrive\문서\스마트팜 전체 작업 폴더입니다.
Android 앱 C:\Users\Admin\OneDrive\문서\스마트팜\android_app APK를 만드는 핵심 폴더입니다.
메인 소스 android_app\src\com\mannal\agri\MainActivity.java 현재 앱 UI와 기능 대부분이 들어 있는 단일 Activity 파일입니다.
매니페스트 android_app\AndroidManifest.xml 앱 이름, 버전, 권한, 시작 Activity를 정의합니다.
빌드 스크립트 android_app\build_apk.ps1 Android SDK 도구로 APK를 직접 컴파일, 서명, 검증합니다.
APK 출력 android_app\out\AgriPrice.apk 최신 기본 APK 산출물입니다.
버전 APK 규칙 android_app\out\AgriPrice_v버전.apk 빌드 시 versionName을 읽어 AgriPrice_v1.4.0.apk 형식으로 생성합니다.

4. Android 앱 현재 기능

시세

  • 관심 품종 3개 입력 및 저장
  • 다음 실행 시 저장 품종 자동 로딩
  • 경기농업기술원 공개 시세 페이지 기반 원문 검색
  • 결과 요약과 표 표시

날씨

  • 관심 지역 3개 입력 및 저장
  • 1번/2번/3번 지역 조회 버튼
  • 기상청 중기예보 원문 기반 농사 작업 참고 문장 생성
  • 일기예보 읽어주기 버튼
  • 음성 명령 오늘 날씨 알려줘 지원

영농일지

  • 날짜, 작목, 작업 내용, 메모 저장
  • 일지 검색
  • 길게 눌러 삭제
  • TXT 백업
  • 음성 명령 일지 메모, 기록해 지원

5. 음성 기능 흐름

날씨 읽기

  1. 앱 시작 후 마이크 권한이 있으면 음성 대기 시작
  2. 사용자가 오늘 날씨 알려줘라고 말함
  3. 앱이 날씨 화면으로 전환
  4. 1번 관심 지역 날씨를 조회
  5. 조회 결과를 TTS로 읽음
  6. 읽는 중 화면을 터치하거나 정지어가 인식되면 중지

영농일지 음성 기록

  1. 사용자가 일지 메모, 기록해, 작업 기록 등으로 시작
  2. 앱이 영농일지 화면으로 전환
  3. 앱이 기록할 내용을 다시 듣는 상태로 전환
  4. 사용자가 작업 내용을 말함
  5. 말한 내용 앞부분은 제목/작목 칸에, 전체 내용은 작업 내용 칸에 입력
  6. 자동 저장

7. 권한과 제약

  • INTERNET: 시세/날씨 공개 페이지 조회에 필요합니다.
  • RECORD_AUDIO: 음성 명령과 음성 일지 입력에 필요합니다.
  • Android 정책상 마이크 권한은 설치/업데이트 시점에 미리 허용할 수 없고, 앱 실행 중 사용자 팝업으로 허용해야 합니다.
  • 일반 APK는 하이 빅스비 같은 시스템 수준 상시 호출어를 만들 수 없습니다. 현재 방식은 앱 실행 중 음성 대기입니다.
  • TTS와 음성 인식을 동시에 쓰면 기기별로 감도가 다릅니다. 날씨 읽기 중 음성 정지는 화면 터치가 가장 확실합니다.

8. 빌드 방법

Windows PowerShell에서 아래 명령을 실행합니다.

cd C:\Users\Admin\OneDrive\문서\스마트팜\android_app
powershell -ExecutionPolicy Bypass -File .\build_apk.ps1

성공 시 기본 APK와 버전 APK가 출력됩니다.

out\AgriPrice.apk
out\AgriPrice_v1.4.0.apk

9. 협업 시 주의할 점

  • 현재 Android 앱은 거의 모든 기능이 MainActivity.java 한 파일에 있습니다. 기능이 더 커지면 화면/음성/저장/네트워크 클래스로 분리하는 리팩터링이 필요합니다.
  • 시세와 날씨는 HTML 원문을 긁어오는 방식이라 외부 사이트 구조가 바뀌면 파싱 품질이 떨어질 수 있습니다.
  • 음성 기능은 실제 스마트폰 기종, Google 음성 인식 상태, 주변 소음, TTS 볼륨에 따라 다르게 동작합니다.
  • 릴리스 전에는 실제 스마트폰에서 마이크 권한, 음성 대기, 날씨 읽기, 일지 음성 저장, TXT 백업을 각각 확인해야 합니다.

10. 다음 개발 추천

  • 영농일지 음성 기록에서 “저장할까요?” 확인 단계를 추가하면 오인식 저장을 줄일 수 있습니다.
  • 일지 제목/작업내용 분리 규칙을 개선해 “제목은 상추 관수, 내용은 오늘 오전 물 줌” 같은 명령을 지원할 수 있습니다.
  • 날씨는 단기예보 API 또는 공식 API 연동으로 온도, 습도, 강수량, 풍속 값을 구조화하는 것이 좋습니다.
  • 시세는 현재 HTML 원문 탐색보다 안정적인 API 또는 크롤링 전용 파서를 분리하는 편이 좋습니다.
  • 앱 내부 설정 화면을 만들어 관심 품종/지역, 음성 대기 사용 여부, 읽기 속도 등을 조절하게 하면 현장 사용성이 좋아집니다.
